Examples

One tag. Every format, source, and surface.

Real, working <movi-player> setups — adaptive streams, multi-track MKV, 360° video, custom byte sources. Load them live or copy the markup straight into your page.

Try it live

Click a poster to spin up a real player. Nothing plays until you ask it to.

Multi-track MKV

Decoded in the browser

A 1080p MKV (Sintel, CC-BY) — multiple audio tracks, embedded subtitles and chapters, all decoded in-browser via WebCodecs + FFmpeg-WASM. Nothing your native <video> tag can see. Open the ⚙ menu.

<movi-player src="movie.mkv" controls thumb showtitle></movi-player>
Adaptive · HLS

HLS stream, auto quality

An .m3u8 ladder that adapts to bandwidth. Same tag, no config — MoviPlayer routes it through its streaming engine automatically.

<movi-player src="stream.m3u8" controls></movi-player>
Adaptive · DASH

MPEG-DASH with captions

A DASH .mpd manifest (H.264 + AAC-LC, so it plays cross-browser incl. Safari) with selectable caption tracks in many languages. Pick one from the ⚙ menu.

<movi-player src="manifest.mpd" controls></movi-player>
Immersive

360° / VR video

Equirectangular video rendered on a sphere — drag to look around, or use the on-screen joystick. Spherical sources auto-enter 360°; here the vr attribute forces it.

<movi-player src="360-clip.mp4" vr vrpad controls></movi-player>
Audio

Music & cover art

Point it at a FLAC/MP3/Opus file — pitch-preserving speed control and stable volume come for free. Embedded album art, when present, becomes an ambient backdrop.

<movi-player src="album/track.flac" controls stablevolume></movi-player>
Split source

Separate video & audio

Video from one file, audio from another — muxed at playback, the way DASH and YouTube ship them. Here a video plays with an independent audio track laid over it.

<movi-player controls showtitle>
  <source src="video-only.mp4" type="video/mp4">
  <source src="audio.m4a" type="audio/mp4" kind="audio">
</movi-player>

Recipes

Copy-paste patterns — same one tag, wired for the setups you supply a source for.

Quality & tracks

Premuxed renditions and multi-language audio, declared as <source> children.

Adaptive

Multi-quality (Auto)

List renditions as <source> children with data-height / data-bandwidth — the player builds a YouTube-style quality menu plus an Auto that adapts to bandwidth. Split audio rides along via kind="audio".

<movi-player controls>
  <source src="4k.mp4"    data-height="2160" data-bandwidth="18000000">
  <source src="1080p.mp4" data-height="1080" data-bandwidth="5000000" data-default>
  <source src="720p.mp4"  data-height="720"  data-bandwidth="2500000">
  <source src="audio.m4a" kind="audio">
</movi-player>
Audio tracks

Multi-language audio

Two or more kind="audio" sources with srclang / label surface an audio-language menu; default (or the page locale) picks the first track.

<movi-player controls>
  <source src="video.mp4">
  <source src="en.m4a" kind="audio" srclang="en" label="English" default>
  <source src="hi.m4a" kind="audio" srclang="hi" label="Hindi">
</movi-player>

Subtitles

Sidecar tracks, styling, and a hook for full ASS/SSA rendering.

Captions

External VTT / SRT

Declare sidecar subtitles the standard way — <track> children (VTT, SRT, TTML). They join any embedded subtitle streams in the ⚙ menu.

<movi-player src="movie.mp4" controls>
  <track kind="subtitles" src="en.vtt" srclang="en" label="English" default>
  <track kind="subtitles" src="es.srt" srclang="es" label="Español">
</movi-player>
Styling

Subtitle look & timing

Tune size, colour, background, and edge, and nudge timing with subtitledelay (seconds; positive = later). All live-updatable at runtime.

<movi-player src="movie.mkv" controls
  subtitlesize="1.3"
  subtitlecolor="#ffffff"
  subtitlebg="rgba(0,0,0,.6)"
  subtitleedge="drop-shadow"
  subtitledelay="0.5"></movi-player>
AdvancedNEW

ASS/SSA subtitles

MoviPlayer decodes to a canvas, so styled ASS/SSA (positioning, karaoke, embedded fonts) can't ride a native <track>. Register a SubtitleRenderer — back it with libass-wasm / jassub — and the player routes the embedded subtitle stream to it: header on configure, each line on pushPacket, a frame on render.

// Adapt any ASS engine (libass-wasm / jassub) to the interface.
class AssRenderer {           // implements SubtitleRenderer
  mount(overlay)              { /* append your <canvas> here */ }
  configure(track, extradata) { /* init libass w/ the ASS header */ }
  pushPacket(pkt)             { /* feed one dialogue line */ }
  render(time, w, h)          { /* draw the frame at w×h */ }
  setDelay(s) {} clear() {} destroy() {}
}

document.querySelector('movi-player')
  .setSubtitleRenderer(new AssRenderer());

Sources & protection

Custom byte sources, authenticated requests, and graceful degradation.

AdvancedNEW

Custom source scheme

Teach the player a scheme once — S3, IPFS, a WebSocket, an encrypted blob, IndexedDB — and any src using it "just works". Your adapter supplies the bytes; the demuxer does the rest.

import { registerSourceAdapter } from 'movi-player';

// register one factory for one or more schemes…
registerSourceAdapter(['s3', 's3a'], ({ url, headers }) =>
  new MyS3Source(url, headers)   // implements SourceAdapter
);
<!-- …then just point src at it -->
<movi-player src="s3://my-bucket/movie.mkv" controls></movi-player>
Auth

Authenticated requests

Attach headers (bearer token, signed cookie) to every media request — the manifest and segments of an HLS/DASH stream and progressive downloads alike. Pass a JSON object as headers.

<movi-player controls
  src="https://cdn.example.com/private/clip.mkv"
  headers='{"Authorization": "Bearer <token>"}'></movi-player>
Resilience

Graceful native fallback

If Movi's WASM pipeline can't read a source (CORS-blocked, opaque, no range support), fallback="native" degrades to a native <video> under Movi's own controls instead of erroring out.

<movi-player src="https://cdn.example.com/clip.mp4"
             fallback="native" controls></movi-player>

Behavior & polish

Attributes that shape playback, look, and controls. Mix and match.

Playback

Resume, seek, thumbnails

thumb scrub-bar previews · resume remembers & restores position · startat opens at a time · fastseek snappier scrubbing.

<movi-player src="movie.mkv" controls
  thumb resume startat="90" fastseek></movi-player>
Appearance

Ambient, poster, theme

ambientmode glow that samples the frame · poster/postertime a still or a frame · theme+themecolor chrome & accent · objectfit scaling · rotate a quarter-turn (0/90/180/270).

<movi-player src="movie.mkv" controls
  ambientmode ambientwrapper="glow-box"
  poster="cover.jpg" postertime="12"
  theme="dark" themecolor="#ff0033"
  objectfit="contain" rotate="90"></movi-player>
Controls

Gestures & decode

playsinline inline playback (no iOS auto-fullscreen) · doubletap double-tap seek ±10s · sw="auto" silent software-decode fallback · buffersize cache window (MB) · nohotkeys off.

<movi-player src="movie.mkv" controls
  playsinline doubletap sw="auto"
  buffersize="256" nohotkeys></movi-player>

API & frameworks

Drive it from JavaScript, or drop the wrapper into React / Vue / Svelte.

JavaScript

Programmatic control

The element mirrors the <video> API and adds media events and accessors — tracks, chapters, cover art, quality.

const el = document.querySelector('movi-player');

el.play(); el.pause(); el.currentTime = 90;       // playback
el.addEventListener('qualitychange', e => …);  // + ended, timeupdate, error
el.player.getChapters();                        // embedded chapters
el.player.getCoverArt();                        // album-art bitmap
el.player.trackManager.getActiveVideoTrack();   // active tracks
Frameworks

React · Vue · Svelte

First-party wrappers ship as subpaths of the one package, passing props straight through to the element — every attribute above works as a prop.

// npm i movi-player   →   movi-player/react · /vue · /svelte
import { MoviPlayer } from 'movi-player/react';

<MoviPlayer src="video.mkv" controls thumb />

Install & embed

Bundle with npm, or drop a single module script from a CDN — no build step.

Getting started

npm or CDN

Importing the element registers the <movi-player> custom element.

# bundlers (Vite, webpack, Next, …)
npm install movi-player
import 'movi-player/element';  // registers <movi-player>
<!-- …or straight from a CDN, no build -->
<script type="module"
        src="https://unpkg.com/movi-player/dist/element.js"></script>
<movi-player src="video.mkv" controls></movi-player>

Every one of these is the same <movi-player> tag — no per-format plumbing, no server transcoding.
